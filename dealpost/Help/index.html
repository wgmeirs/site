A - Description - top

This script Bootsrap Form Pack allows you to obtain a quick & easy forms for your website. You only need to put this script into any directory on your hosting account, edit configuration file and give the link to your customers.

The pack has a set of solutions that allow you to create:
- Contact Form (3 forms)
- Booking / Event Form (2 forms)
- Upload file (2 forms)
- Newsletter subscribe form
- Register form (2 forms)
- Login form
- Address form

Demo http://www.beaenea.com/envato/codecanyon/bootstrap_form_pack/1_contact_form_1.php

B - Installation - top

1- Select the form that you want to use. Open "config.php" (into /includes)and edit the values ​​that you want.

2- Upload the files (including the folders /includes and /js) on your server ... and you will be the form ready for use.

3- if you use the email log into the database, you'll have to use the file before anything install_mysql.php.

4-At the beginning of the file has to contain have the following: 

<?php 
require_once "includes/config.php";		    
require_once "includes/inc.formpack.php"; 	
?>



C - Date Pickers - top

The datepickers have the following options:

Name type default description
format string 'mm/dd/yyyy' the date format, combination of d, dd, m, mm, yy, yyy.
weekStart integer 0 day of the week start. 0 for Sunday - 6 for Saturday
startDate date null The earliest date that may be selected; all earlier dates will be disabled
endDate date null The latest date that may be selected; all later dates will be disabled
autoclose boolean false Whether or not to close the datepicker immediately when a date is selected
startView string 0, 'month' The view that the datepicker should show when it is opened. Accepts values of 0 or 'month' for month view (the default), 1 or 'year' for the 12-month overview, and 2 or 'decade' for the 10-year overview. Useful for date-of-birth datepickers
keyboardNavigation boolean true Whether or not to allow date navigation by arrow keys
language string en The two-letter code of the language to use for month and day names. These will also be used as the input's value (and subsequently sent to the server in the case of form submissions). Currently ships with English ('en'), German ('de'), Brazilian ('br'), and Spanish ('es') translations, but others can be added (see I18N below). If an unknown language code is given, English will be used


D - Add new fields - top

To add an extra field you only have to put the field inside the <form> automatically Bootstrap system Form Pack your cares and automatic process.
It is important not to forget to put the property name and id.
such as: <input type="text" name="whatyouwan" id="whatyouwan">

E - Validation - top

To validate the fields just add the class validate[required]
for email validate[required,custom[email]]
for url validate[required,custom[url]]
for phone validate[custom[phone]]
for numbers validate[required,custom[onlyNumberSp]]
for only letters validate[required,custom[onlyLetterSp]]
for IP validate[required,custom[ipv4]]

The validation system pages has to contain:
-Link to js and css 

<link rel="stylesheet" href="js/jQuery-Validation-Engine-master/css/validationEngine.jquery.css" type="text/css"/>
<script src="js/jQuery-Validation-Engine-master/js/languages/jquery.validationEngine-en.js" type="text/javascript" charset="utf-8"></script>
<script src="js/jQuery-Validation-Engine-master/js/jquery.validationEngine.js" type="text/javascript" charset="utf-8"></script>
-Validation init
<script>
jQuery(document).ready(function(){
	jQuery("#form").validationEngine();
	});
</script>

Uploads limitations Sizes & Extensions

For size file limitations apply this into form tag ( where value its max size in bytes)

<input type="hidden" name="MAX_FILE_SIZE" value="1000000" />

Allowed types

In includes/inc.formpack.php in class formpack

$upload->setAllowed( array('types'=>array('png','jpg','gif','pdf','txt','doc')) ); // set extensions allowed

F - Token, the anticaptcha - top

To fight spam using a system based on tokens.
Apply into form: 

<?php echo formtoken::getField() ?>

Validate 
<?
if (formtoken::validateToken($_POST)) {
// do send
}
?>
